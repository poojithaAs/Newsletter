🔍 Why This Matters

Managing dozens of YAML files across environments is painful. Small changes. Big confusion. This edition helps you escape config chaos using better structure, templates, and tools like Jsonnet.

📚 What You'll Learn

1. Two Common Config Problems

  • Replication Toil – Copying the same config changes everywhere.

  • Complexity Toil – Struggling to manage overengineered config setups.

2. Why YAML Becomes a Hidden Programming Language

  • YAML with templating (Jinja, Go templates) quickly becomes unreadable.

  • Updates are hard. Reuse is missing.

3. Top 5 Mistakes to Avoid

🚫 Not treating config as code

🚫 Adding features ad hoc

🚫 Optimizing for just one use case

🚫 Mixing config with side effects

🚫 Using general-purpose languages like Python/Ruby

🔧 Best Practices for Your Apps

✅ Let the config language define structure

✅ Let your app focus on behavior

✅ Use named objects, not arrays

✅ Group related configs together

✅ Avoid logic in strings — do it in Jsonnet instead

🧪 Test & Validate

  • Use to test templates

  • Add pre-commit checks and validations

  • Validate generated output, not just source

  • Never ignore unknown fields — they're usually typos

⏱️ When to Process Config

  • Before Commit – Best for reviews and rollbacks

  • At Build Time – Ideal for CI/CD pipelines

  • At Runtime – Only if sandboxed and safe

🛡️ Protect Against Bad Config

Even config can cause outages: infinite loops, memory issues, or crashes.

✅ Use or safe wrappers

✅ Never evaluate untrusted config code directly

✅ Jsonnet's Go version adds safety

✅ Summary – Configuration Made Simple

  • Start simple. Stay consistent.

  • Use abstraction early to avoid repetition

  • Pick the right tool — Jsonnet > raw YAML

  • Test and validate just like real code

  • Good config = reliable systems
