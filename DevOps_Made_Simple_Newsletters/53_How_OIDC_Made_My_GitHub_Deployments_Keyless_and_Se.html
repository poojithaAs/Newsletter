How OIDC Made My GitHub Deployments Keyless and Secure

View on LinkedIn

✍️ By Poojitha A S — DevOps Engineer | SRE | Cloud Automation Lead

"Security doesn't mean adding more locks , It means removing the keys."

🔐 The Problem: Static Access Keys Were Never Safe

For years, CI/CD pipelines relied on fixed AWS keys stored in GitHub Secrets. But they had hidden dangers:

  • They never expired automatically.

  • If a repo was exposed, so were the keys.

  • Rotation was manual and error-prone.

  • Once stolen, keys stayed valid until manually revoked.

In short: secrets were the weakest link. A single leaked key could unlock your entire AWS account.

🔑 The Fix: OIDC (OpenID Connect)

OpenID Connect replaces static credentials with identity-based access. Instead of storing permanent AWS keys, GitHub proves its identity in real-time to AWS.

Here's what happens behind the scenes:

  1. GitHub requests a token from its OIDC provider.

  2. AWS verifies that token against the GitHub identity provider you've configured.

  3. AWS STS issues temporary keys — valid only for ~15 minutes.

  4. Terraform deploys with those short-lived credentials.

No long-term keys. No secrets stored. No rotations needed.

🧠 How It Works in Your Workflow

Your GitHub Actions pipeline requests an OIDC token during runtime. AWS confirms the request using a trust policy tied to your repo and branch. Once verified, it grants temporary IAM role access to perform deployments.

📊 Old vs. New Authentication
Why It Matters
OIDC shifts your security posture from secret management → identity 
Here's what your team gains:
✅ No stored keys → Nothing to leak
✅ Auto-expiring tokens → No rotation needed
✅ Full audit trail → CloudTrail logs every action
✅ Granular access → Repo & branch–specific trust policies
✅ Zero-trust ready → Meets SOC 2 / ISO 27001 / FedRAMP standards
🤩 Design for Temporariness
Every DevOps pipeline should be built around short-lived access. This way, even if something breaks, nothing dangerous persists.
"In modern security, the safest secret is the one that doesn't exist."
💡 Key Takeaways
1. Stop storing AWS keys — replace them with OIDC trust.
 2. Restrict roles to specific repositories and branches.
3. Let Terraform + GitHub handle auth dynamically. 
4. Use CloudTrail to track every OIDC authentication. 
5. Build zero-trust pipelines by design.
