"If you can't explain Kubernetes networking simply, you don't understand it deeply."

1. What is a Pod?

  • Smallest unit in Kubernetes.

  • Each pod has its own IP address inside the cluster.

2. What is a Service?

  • A stable virtual IP that lets you access pods — even if the pods change.

  • Types: , , , .

3. What is an Endpoint?

  • A list of pod IPs behind a Service.

  • Think: Service is the front door, Endpoints are the actual pods.

4. What is an EndpointSlice?
  • A scalable version of Endpoints.
  • Behind the scenes, Services now use EndpointSlices.
  • More efficient for big clusters.
5. What is CoreDNS?
  • The cluster's internal DNS server.
  • Turns → Pod IP
CNI (Container Network Interface)
Kubernetes doesn't create networking itself. It asks a CNI plugin to:
  • Assign pod IPs
  • Connect pods via bridges/routing

kube-proxy: The Silent Router
