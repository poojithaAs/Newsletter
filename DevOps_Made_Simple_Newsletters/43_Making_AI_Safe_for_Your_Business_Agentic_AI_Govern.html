Making AI Safe for Your Business: Agentic AI Governance

View on LinkedIn

✍️ By Poojitha A S | #DevOps #SRE #AI #Governance

If you want the vendor's insurance, keep the guardrails on—and keep the receipts.

🔙 The Story (What Really Happened)

Last quarter, a team built an internal AI tool to draft FAQs, emails, and code fixes. It worked brilliantly—until Legal found two problems in one week:

  • Some code looked too close to GPL-licensed code.

  • A customer FAQ was almost identical to a published article.

Engineering said, "Copilot wrote it." Product said, "We moved fast." Legal asked: "Are we protected if there's a lawsuit?"

Silence.

🔙 The Problem (Why Companies Get Into Trouble)

Having rules isn't enough—you need proof. Vendor protection (indemnity) only applies if conditions are met

  • GitHub Copilot can block code suggestions that match public GitHub repos,

  • AWS offers full indemnity for certain services (e.g., Titan, CodeWhisperer Pro). 【AWS】

  • Google Cloud indemnifies outputs only from listed Generative AI services. 【Google Cloud】

  • Microsoft 365 Copilot content can be stored in a hidden mailbox folder for retention/eDiscovery—but only if you set it up.Without these switches ON, you're exposed.

🔙 What's at Stake (Why This Matters)

Indemnity is like car insurance:

  • Seatbelt on = guardrails ON.

  • Paperwork = audit logs.

  • Drive responsibly = follow licenses & usage rules.

Turn off guardrails or skip logging → the bill is yours, not the vendor's.

🔙 The 4-Step Fix

1. Know Who's Responsible

  • Microsoft: Use paid Copilot/Azure OpenAI with safety ON. Save screenshots.

  • AWS: Stick to indemnified tiers (Titan, CodeWhisperer Pro). Track non-covered models.

  • Google Cloud: Confirm services are on the indemnified list before launch.

2. Set Company-Wide Safety Switches

  • In GitHub Copilot Enterprise → Turn ON "Block suggestions matching public code."

  • If you allow matches, enforce references + licensing compliance. Audit quarterly.

3. Make AI Content Compliant

  • Route Copilot outputs to retention/eDiscovery systems. Test retrieval.

  • Treat AI code like third-party code → run SCA/SAST/license scans before merge.

4. Keep the Receipts

  • Log model, version, prompts, safety flags, approvers, eval reports.

  • Track all exceptions: who approved, why, for how long.

🔙 Field Example

Last sprint, with duplicate-match blocking ON, Copilot flagged 3 of 47 code suggestions. One matched GPL. Stopped before merge = no legal fire drill.

Bonus: Copilot chats were found instantly during a compliance audit.

🔙 Do / Don't (Leader's Cheat Sheet)

✅ Do This

  • Turn ON public code blocking (or enforce references).

  • Use indemnified services (Microsoft CCC, AWS Titan/CodeWhisperer Pro, Google's covered list).

  • Enforce CI gates: license, SCA, SAST.

  • Configure Copilot logs for retention/eDiscovery.

❌ Don't Do This

  • Paste referenced code without attribution.

  • Turn off guardrails and assume coverage.

  • Store copyrighted web content in your knowledge base.

🔙 TL;DR

AI governance isn't a slide—it's settings + logs + ownership. Use indemnified services, keep guardrails ON, enforce compliance scans, and save receipts.

So when Legal asks "Are we covered?" you can confidently say: "Yes—here's the proof."

🔙 References

  • Microsoft: Copilot Copyright Commitment (CCC).

  • Microsoft Learn: Purview retention/eDiscovery for Copilot.

  • GitHub Docs: Public code blocking (~150-char match), Code Referencing.

  • AWS: Indemnity for Titan & CodeWhisperer Pro.

  • Google Cloud: Indemnified Generative AI services.
