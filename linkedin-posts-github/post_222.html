#DAY205 Most people know terraform apply, but here&#x27;s what really saved my team last time You know terraform apply. But have you ever used: terraform taint …to force a resource to be recreated — without touching the code? I had a weird issue. An EC2 instance looked fine in code, but something was off in runtime.

Instead of deleting manually, I ran: terraform taint aws_instance.my_instance terraform apply Boom — fresh resource, clean state, zero manual patching. And here&#x27;s why this matters:

• It&#x27;s safer than deleting from AWS console

• You stay fully declarative

• You let Terraform take care of dependencies Tip: If you&#x27;re using Terraform in teams, learn these too:

• terraform import — bring existing infra under code

• terraform state mv — move resources between modules

• terraform workspace — for managing multiple environments These are small tools, but they change how you think as a DevOps engineer. What&#x27;s a Terraform command that you wish more people used? #Terraform #DevOps #InfrastructureAsCode #CloudEngineering #AWS #IaC #TerraformTips #DevOpsMindset
