#WeBrokeItTogether: Reliability for AI Episode 022426: The Config File That Ate the Internet Based on the Cloudflare outage on November 18, 2025

The #Setup A company runs AI powered bot detection across its entire global network. Every web request. Billions per day. Each one was scored by a machine learning model. Human or bot. The model reads a settings file that updates every five minutes. It has worked perfectly for years. Then one morning, half the internet goes down.

Sara Baqla (#Dev): Users are reporting errors on X, Spotify, ChatGPT, and Shopify. Are we under attack? Poojitha A S (#DevOps #SRE) That was the first guess. But no. We did this to ourselves.

Sara Baqla: How?

Poojitha A S: A routine database permission change altered how a query joined two tables. That query builds the settings file for the ML bot detector. Before the change, it returned about 60 rows. After the change, it returned duplicates. Over 200 rows.

Sara Baqla: So the file got bigger?

Poojitha A S: Yes. And the system had a hardcoded limit of 200 items. It had been running at 60 for years. Nobody expected it to hit

200. When it did, the code did not degrade gracefully. It crashed. On every server. Globally.

Sara Baqla: Every server?

Poojitha A S: The config pushes to the entire network every five minutes. So every five minutes, every server loaded a broken file, crashed, restarted, loaded it again, and crashed again.
