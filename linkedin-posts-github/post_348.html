#Day120 Advanced Terraform Troubleshooting Commands

1. Debug Logs (TF_LOG) #Command: export TF_LOG=DEBUG export TF_LOG_PATH=terraform_debug.log

• Use:

• Enable detailed logs to trace API calls, provider interactions, and errors. 2. terraform state mv #Purpose: Moves resources between modules or state files. #Command: terraform state mv <source> <destination> #Use:

• Fix state misalignment when resources are relocated in configurations. 3. terraform state replace-provider #Purpose: Updates provider references in the state file.

• Command: terraform state replace-provider <old_provider> <new_provider> #Use:

• Migrate resources to a new provider (e.g., hashicorp/aws to custom/aws). 4. terraform console #Purpose: Interactive REPL to evaluate HCL expressions. %Command: terraform console #Use:

• Test variable values, outputs, and interpolations. 5. terraform force-unlock #Purpose: Unlocks a state file locked by a failed operation.

• Command: terraform force-unlock <LOCK_ID> #Use:

• Resolve lock conflicts in remote backends like S3 or Consul. 6. terraform workspace #Commands:

• terraform workspace list: View all workspaces.

• terraform workspace show: Identify the active workspace.

• terraform workspace select <name>: Switch to a specific workspace. #Use:

• Troubleshoot workspace-specific configurations and state isolation. 7. terraform providers #Purpose: Lists all providers used in your configuration. #Command: terraform providers #
