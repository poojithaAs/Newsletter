#Day106 Auto Scaling in AWS is a feature that automatically adjusts the number of compute resources, such as EC2 instances, in response to demand. It helps optimize performance, reduce costs, and ensure that applications remain available and responsive. Here&#x27;s an overview of how it works: Key Components of AWS Auto Scaling

1. Auto Scaling Group (ASG):

• A group of EC2 instances managed together.

• Ensures the group maintains the desired number of instances.

• Can scale the number of instances up or down automatically based on conditions.

2. Launch Configuration or Launch Template:

• Specifies the configuration details for the EC2 instances to be launched, including:

• Instance type

• AMI ID (Amazon Machine Image)

• Security group

• Key pair

3. Scaling Policies:

• Rules that define when to scale in (reduce instances) or scale out (add instances).

• Triggered by monitoring metrics like CPU utilization, memory usage, or custom CloudWatch metrics.

4. CloudWatch Alarms:

• Monitors specific metrics and triggers actions based on thresholds (e.g., CPU usage > 70%).

• Works with scaling policies to dynamically adjust capacity. Types of Auto Scaling

1. Dynamic Scaling:

• Automatically adjusts the number of instances based on demand.

• Uses policies tied to metrics like CPU utilization or request count.

• Example:

• Scale out when CPU utilization exceeds 70%.

• Scale in when CPU utilization falls below 30%.
