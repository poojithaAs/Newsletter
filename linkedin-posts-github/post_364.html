#Day107 Preparing for Tomorrow&#x27;s Kubernetes HA Cluster Demo We&#x27;re excited to dive into setting up a High Availability Kubernetes Cluster tomorrow with 3 master nodes and 2 worker nodes. Today, let&#x27;s ensure we&#x27;re ready with the prerequisites: Pre-Requisites

1. Machines:

• 5 machines (or VMs) with:

• OS: Ubuntu 20.04 (or compatible)

• Minimum: 2 CPUs, 4GB RAM, 20GB storage (per node)

2. Network Setup:

• Private IP Range: Ensure a non-overlapping CIDR block for your internal VPC/subnet. Example:

• VPC CIDR: 10.0.0.0/16

• Subnet 1: 10.0.1.0/24 (Master Nodes)

• Subnet 2: 10.0.2.0/24 (Worker Nodes)

• Assign static or reserved private IPs to each node.

3. Public IP or Load Balancer:

• For control plane HA, set up a load balancer (e.g., HAProxy, AWS ELB) with a virtual IP accessible by all nodes.

4. Networking Plugin (CNI):

• Decide on a plugin like Calico or Flannel for Pod networking. We&#x27;ll use Calico in the demo.

5. Firewall Rules:

• Open these ports:

• API Server (6443): For communication between nodes.

• etcd (2379-2380): For master node communication.

• Node Communication (10250-10255): For kubelet and metrics.

• Custom Ports: Based on applications/services.

6. DNS Configuration:

• Ensure DNS resolution works between all nodes (private DNS or host entries).

7. Hostnames:

• Set distinct hostnames for each node (e.g., master-1, master-2, worker-1). Required Software
