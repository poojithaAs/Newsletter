#WeBrokeItTogether RAG Series Episode 022026 : The Chatty JSON The Setup: Sara built a dashboard that visualizes server stats. It relies on the RAG system outputting raw JSON. Suddenly, the dashboard crashes. The logs show the LLM output: "Sure! Here is the JSON you asked for: { ... } Hope this helps!"

Sara Baqla(Dev): "Why is it so polite?! I told it to ONLY output JSON!"

Poojitha A S (DevOps): "It&#x27;s a chat model, Sara. It likes to chat."

Sara Baqla: "I&#x27;m writing a Regex to strip the text. Or maybe I&#x27;ll threaten it in the prompt. &#x27;If you speak, I will delete you.&#x27;"

Poojitha A S: "Please don&#x27;t threaten the AI. Also, Regex is fragile. If the model changes its greeting, your code breaks again."

Sara Baqla: "Then how do I shut it up?"

Poojitha A S: "Stop asking it nicely in the prompt. Force the format at the API level."

Fix: Poojitha switched the API call to use JSON Mode (or Function Calling). This forces the model to output only a valid JSON object, treating the instruction as a structural constraint rather than a suggestion.

Result: The "polite" conversational filler vanished. The dashboard is now 100% stable. RAG Failure Point #5: Wrong Format. Don&#x27;t rely on a probabilistic model for deterministic formatting. Separate your formatting instructions or use constrained decoding. #DevOps #SRE #RAG #SWE #VectorDatabase
