#DAY204 #TerraformTales: The Mystery of the Missing S3 Bucket Problem: You apply your Terraform config, it runs successfully. But the S3 bucket? Nowhere to be found. You check the state. You double-check the region. You triple-check the bucket name. Still… nothing.

Turns out—you used a count to conditionally create the bucket: resource "aws_s3_bucket" "logs" { count = var.enable_logging ? 1 : 0 bucket = "my-app-logs" } But in your variables? enable_logging = false. So Terraform silently skips creating the bucket. No errors, no warnings, just… nothing.

Solution: Add output visibility or a warning to avoid getting ghosted by your own config: output "logging_enabled" { value = var.enable_logging description = "Is logging enabled for this environment?" } Or, better yet—use terraform plan and actually read it (yes, we&#x27;re all guilty). Lesson: Just because your plan is green doesn&#x27;t mean your infra is real.

Terraform will always do exactly what you tell it—nothing more, nothing less.
