#Day121 Understanding #DaemonSets in Kubernetes DaemonSets ensure specific pods run on all (or specific) nodes in a Kubernetes cluster. They are ideal for node-level operations like monitoring, logging, and security, ensuring uniform functionality across the cluster. Where Are #DaemonSets Used?

1. #Monitoring

• #NodeExporter: Collect system-level metrics like CPU, memory, and disk usage for Prometheus.

• OpenTelemetry Collector: Gather distributed traces, metrics, and logs for application performance monitoring and send them to tools like Jaeger or Grafana.

• Why DaemonSet? To ensure telemetry data is collected consistently from all nodes.

2. #Logging

• #Fluent Bit: Collect and forward logs to Elasticsearch, Loki, or Splunk.

• #Filebeat: Ship logs from nodes to centralized logging platforms like ELK stack.

3. #Security

• Falcon Detect runtime threats by monitoring system calls.

• #Kube-bench: Perform Kubernetes security benchmark checks on all nodes.

4. #Networking

• Cilium Enforce advanced network policies and observability using eBPF.

• Node-local #DNS Cache: Improve DNS resolution for high-performance applications.

5. #ApplicationPerformanceMonitoring (APM)

• Dynatrace OneAgent: Monitor applications, processes, and infrastructure.

• Why DaemonSet? Dynatrace requires agents running on all nodes to capture full-stack telemetry and detect anomalies.
