From 45-Minute Builds to 2.5 Minutes: How We Fixed Our Flaky CI Sara Baqla hit "Rerun" for the third time. Same code. Different failure. The team had stopped trusting our tests, and honestly, who could blame them?

Sara Baqla: "Rerun #3. Still red."

Poojitha A S: "Same test failing?"

Sara Baqla: "Nope. Different one every time. 45 minutes per run. 8% flake rate."

Poojitha A S: "So people started merging without waiting?"

Sara Baqla: "Yep. The green checkmark became... optional." The #Investigation

Sara Baqla: "62 tests are gaslighting us."

Poojitha A S: "What&#x27;s causing it?"

Sara Baqla: "Shared database, race conditions, 35 minutes of sleep() commands."

Poojitha A S: "We&#x27;re paying cloud bills for robot naps?" The #Fixes

1. Parallel Test Groups (Not Just Tests)

Poojitha A S: "Reddit was right—don&#x27;t parallelize tests, parallelize groups." API tests → 4 workers DB tests → 3 workers Integration → 2 workers Each group gets its own container

Sara Baqla: "45 min → 12 min just from this."

2. Quarantine the Flaky Ones

Sara Baqla: "What if flaky tests... just don&#x27;t block deploys?"

Poojitha A S: "Controversial. I love it." Main pipeline: Stable tests only Quarantine pipeline: Flaky tests notify, don&#x27;t block Rule: Fix in 1 sprint or delete

Sara Baqla: "BuildPulse caught the flaky 8%. We quarantined them, fixed root causes."

3. Fresh Database Every Time

Poojitha A S: "TestContainers gives each group a temporary container."

Sara Baqla: "Zero pollution. Finally."

4. Wait Smart, Not Blindly

Sara Baqla: "Replaced sleep(5) with 50ms polling."

Poojitha A S: "5s → 0.34s average wait."

5. Test Only What Changed Frontend tweak → 23 tests (was 800) Logic change → 127 tests README → 0 tests

6. Remote Caching

Poojitha A S: "Bazel shares results across PRs. 73% cache hit rate."

Sara Baqla: "Most tests finish instantly now." The #Results ⏱️ 45 min → 2.5 min (18x faster) 🎲 8% flake → 0.03% 🚀 2 deploys/day → 23/day 💚 62 flaky tests → 0 blocking PRs The Controversial Take

Poojitha A S: "We don&#x27;t fix flaky tests anymore."

Sara Baqla: "Wait, what?"

Poojitha A S: "We quarantine them. If a test can&#x27;t be stable in 1 sprint, it gets deleted or moved off the critical path."
